# `data/generatable/`

Nothing in this folder is committed to GitHub (see the repository `.gitignore`);
this README is the only tracked file here. Everything else is **generated by the
scripts in `scripts/`** from the small committed inputs in `data/provided/` plus
the large external assets documented in `data/externalsourcerequired/` (raw
images, genotype marker files, model weights).

By default every output-producing script writes here. Each entry below lists the
default path, the script that produces it, and what it needs as input.

## Files and how to regenerate them

### `embeddings.npz` / `embeddings_summary.csv` — `scripts/extract_embeddings.py`
Per-crop embedding matrix (`float32`) plus metadata, with an automatic per-image
summary sidecar named from the output stem. Needs an image-list CSV (the committed
`data/provided/examples/example_image_list.csv`, or the full image set from
`data/externalsourcerequired/`) and the SAM3 or DINOv2 weights in
`data/externalsourcerequired/`. Default crops are `2016 x 2016` pixels and are
OpenCV area-resized to `1008 x 1008` before either backend. On the full dataset
this takes ~10-12 hours.

```bash
python scripts/extract_embeddings.py data/provided/examples/example_image_list.csv --backend sam3
```

### `dimreduction/` — `scripts/calculate_pcs_ics.py`
`pc_scores.csv`, `ic_scores.csv`, `pca_variance_curve.csv`, `test_groups.csv`,
`run_metadata.json`, and saved `models/*.joblib`. Needs an embeddings table.

```bash
python scripts/calculate_pcs_ics.py --embeddings data/generatable/embeddings.npz
```

### `blues/` — `scripts/calculate_blues.py`
`blues_<environment>.csv`, `heritability_<environment>.csv`, and
`variance_partitioning_<environment>.csv`. Needs a score table (IC scores or
embeddings) and the committed `data/provided/field_image_metadata.csv`.

```bash
python scripts/calculate_blues.py --scores data/generatable/dimreduction/ic_scores.csv --environment all --include-leaf-area
```

### `random_forest/` — `scripts/train_random_forest.py`
`rf_predictions.csv`, `rf_image_predictions.csv`, `rf_genotype_predictions.csv`,
`rf_fold_accuracy.csv`, `rf_overall_accuracy.csv`, `rf_genotype_accuracy.csv`,
`rf_feature_importances_by_fold.csv`, `rf_feature_importance_summary.csv`. Needs a
feature table plus the committed `data/provided/human_disease_scores.csv` /
`exg_ratings.csv` / `field_image_metadata.csv`. Use a distinct `--out-dir` per
target so runs do not overwrite each other.

```bash
python scripts/train_random_forest.py --features data/generatable/dimreduction/ic_scores.csv --target human_score --out-dir data/generatable/random_forest_human
```

### `ic_disease_correlation/ic_human_score_correlation.csv` — `scripts/correlate_ics_disease.py`
Per-IC Spearman correlation with the human disease score (pooled and per
environment). Needs IC scores and the committed
`data/provided/human_disease_scores.csv`.

```bash
python scripts/correlate_ics_disease.py --ic-scores data/generatable/dimreduction/ic_scores.csv
```

### `gwas/` — `scripts/run_gwas_panicle.py`
`traits/<trait>_marker_pvalues.csv`, `plots/<trait>_*.png`, `gwas_summary.csv`,
`significant_markers.csv`, `top_markers.csv`, `effective_tests.json`,
`run_metadata.json`. Needs a BLUE file plus the genotype marker VCF/PLINK in
`data/externalsourcerequired/vcf/` and the committed
`data/provided/gwas_covariates_leaf_area_flowering_time.csv`.

```bash
python scripts/run_gwas_panicle.py --blue-file data/generatable/blues/blues_all.csv \
  --covariate-file data/provided/gwas_covariates_leaf_area_flowering_time.csv \
  --covariate-cols mask_pixels_blue,days_to_flower_blue --drop-missing-samples
```
